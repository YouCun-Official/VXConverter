🚀 推荐开发顺序（性价比优先）
Tier 1 (秒杀)：二维码、电子签名、图片压缩、证件照。
理由：这4个可以快速填满“其他工具”和“多媒体”板块，用户感知强，开发快。
Tier 2 (核心)：PDF合并/拆分/水印。
理由：PDF操作是刚需，且 Node.js 生态有较好的库（如 pdf-lib），云函数能撑住。
Tier 4 (可接入API)：AI文档纠错。
理由：虽然技术难，但接 API 很简单（如 deepseek/kimi/openai），只需付 token 费。
Tier 3/4 (慎重)：Office文档互转、视频处理。
理由：建议先放放，或者直接找现成的 API 服务对接，不要自己写转换逻辑。
